## Summary

Create a GitHub Actions workflow that automatically builds and deploys the Miti calendar website to GitHub Pages on every push to the main branch. The workflow will compile TypeScript, build with Vite, and deploy the dist folder using GitHub's native Pages deployment mechanism. This eliminates manual deployment steps while protecting production through build validation.

## Phase 1: Design & Artifacts

### 1.1 Data Model

This feature does not involve data modeling. It is pure infrastructure automation.

### 1.2 Workflow Design

**Artifact**: `.github/workflows/deploy.yml`

**Workflow Steps**:
1. Trigger: Push to main branch
2. Setup: Node.js environment
3. Build: Run `npm install` (cache dependencies)
4. Build: Run `npm run build` (TypeScript + Vite)
5. Deploy: Push dist/ folder to GitHub Pages using `actions/deploy-pages@v2`
6. Logging: GitHub Actions automatically logs all step outputs

**Implementation Pattern**: Standard GitHub Actions workflow using official `actions/setup-node`, `actions/upload-artifact`, and `actions/deploy-pages` actions.

### 1.3 Deployment Target Configuration

**Prerequisite**: Repository must have GitHub Pages enabled in settings (assumed in spec Assumptions section).

**Deployment Method**: Using `actions/deploy-pages@v2` action:
- Automatically uses GitHub-provided token for authentication
- Deploys to official GitHub Pages service
- Handles GitHub's Pages URL routing

### 1.4 Contracts

**GitHub Actions Workflow Contract**:
- **Input**: Push event to `main` branch
- **Process**: Build using existing npm commands
- **Output**: Deployed site at `https://<owner>.github.io/<repo>/` (or custom domain if configured)
- **Success**: Site accessible and updated within 5 minutes
- **Failure**: Build errors prevent deployment; logs available in GitHub Actions tab
